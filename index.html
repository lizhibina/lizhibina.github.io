gradle和maven区别

Posted on 2019-04-13

gradle和maven区别

1.Gradle和Maven都是项目自动构建工具，虽然两者都是项目工具，但是maven现在已经是行业标准，Gradle是后起之秀，很多人对他的了解都是从android studio中得到的，Gradle抛弃了Maven的基于XML的繁琐配置，众所周知XML的阅读体验比较差，对于机器来说虽然容易识别，但毕竟是由人去维护的。取而代之的是Gradle采用了领域特定语言Groovy的配置，大大简化了构建代码的行数

2.maven使用.pom文件，基于XML，gradle使用.gradle文件，基于Groovy，gradle去掉maven中需要固定配置的groupId，artfactId，package等基本参数，通过 apply plugin: ‘java’，apply plugin: ‘war’来表示打包方式，也提供sourceCompatibility，version等一些实用的标签。

3.maven通过groupId、name和version到配置的Repository里寻找指定依赖中的Jar包。Gradle的生命周期更为复杂，但也支持Maven依赖中的构建生命周期，compile、runtime、testCompile和testRuntime分别对应项目不同阶段的依赖。 并且还简化maven的配置命令。

gradle方式 更为简洁

compile(‘org.springframework:spring-context:2.5.6’)

搭建hexo博客（github仓库版）

Posted on 2019-03-11

搭建个人hexo博客(mac)

环境安装

安装node.js(依赖node.js) http://nodejs.org/

安装Git

安装Hexo

安装淘宝镜像cnpm

安装教程

打开终端 查看node.js/git/npm是否安装成功

切换root用户下操作

安装淘宝镜像

$ npm install -g cnpm –registry=https://registry.npm.taobao.org

安装hexo

cnpm install -g hexo-cli

hexo -v验证是否成功

创建blog文件夹

进入blog文件夹里执行 sudo hexo init(初始化hexo)

启动hexo: hexo s

报错：ERROR Try running: ‘npm install hexo –save’

执行cnpm install 和hexo server

blog下执行hexo n “第一篇博客”

blog下hexo clean 清理一下

blog下hexo s

blog下hexo g 生成博客

hexo d上传到github上

把博客部署到github上

在blog安装git部署的插件 cnmp install –save hexo-deployer-git

创建github仓库

在blog下的_config.yml中配置github仓库地址

deploy:

type: git

repo: https://github.com/lizhibina/lizhibina.github.io.git

branch: master
